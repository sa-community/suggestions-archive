Currently, if you try to delete a "define" block while there are still uses of it in the project, you get a popup saying "To delete a block definition, first remove all uses of the block." This addon would change it so it would behave like deleting a variable; you'd get a confirmation if the block was used more than once and then, if you confirm, the block and all its uses would be deleted for you. I think the devs who make bigger projects will especially like this one because the current system is a royal pain for big projects where an orphaned use of the block could be anywhere, but I think everyone will appreciate this and find it useful.
