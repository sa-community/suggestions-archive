When I make folder for costumes, I do that to save space and make it look nicer and more organised. But whenever I have a costume inside of a collapsed folder selected, and go from the code tab to the costumes tab (or switch to that sprite), it opens the folder that the selected costume is in! And that really annoys me, because I like having them closed unless I need to edit stuff in them. I propose 2 changes to solve this:
1. Do not automatically open the folders when switching to the costumes tab, but then this means that you don't intuitively know which folder it's in so..
2. Show the currently selected folder (the folder which contains the selected costume) in the same way as costumes themself (the solid blue/red[for turbowarp] colour around the sprite name and resolution), because currently there is no way to see what folder a costume is in without looking at the costume name.
