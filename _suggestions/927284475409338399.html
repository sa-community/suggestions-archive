The drag and drop feature wasn't the point.
You're missing the point.
The (although probably not that common)problem that I think my suggestion can fix is:
What if I have a huge thing of data that I want to put into my project and be able to reload by pressing the add to list button again but I don't want to create em by one.
What if I could just right click - import - set my list and separation settings - and then be able to load it all in at once by putting it in a custom block and then running it.
(And I guess I could in this (possibly unrealistic)scenario create a second list and then copy that list over using a big loop but I still think the other way is better.)

Actually here let me give you an example:
You are making game which uses a 2d list grid with each row separated by commas and each column separated by lines.
To load in a level you simply load the level into a list and then draw the list as tiles.
However when importing a level you don't want to create a temp list for each separate level.
What if instead you make your level, save it to a list, export the list, then drag and drop it into scratch as a bunch on add to list blocks.
So there is at least some value to this idea and I still think it should be implemented.
