When you drag a hat block in from the palette (for example, "when clicked") , scratch generates a random ID internally for that block. If there are multiple of the same hat block in the same sprite, scratch needs to choose which of them will run first. The order is determined by the ID that it randomly picked, essentially meaning each hat block has a hidden random "priority" which determines in what order it will run relative to other hat blocks of the same type. I suggest an addon which allows you to view and modify this normally hidden priority to change the order of execution of the hat blocks.
Here's how I think it could work: the addon would look at the IDs of all the hat blocks of the same type in the sprite and sort them the same way scratch does. This would allow it to assign each hat block a simple number starting from 1 which indicates the "priority" of that hat block. This number could be shown in the context menu for that hat block, with two arrows to increase or decrease its priority. When the priority is changed the IDs of the swapped hat blocks are swapped. So if they priority changes from 3 to 4, the IDs of the hat blocks with priority 3 and 4 are swapped, thus swapping their priority.
I might do this one day but probably not soon so I thought I'd put it in here to see what people think and in case someone else wants to work on it.
