.txt Files are Being Treated as .csv Files When Imported Into List
When you want to import a file from your computer into a list, if you choose a .txt (plain text) file, scratch acts as though it is a .csv (comma separated values) file.
To show this I've created this .txt file. It should be imported so that each line becomes an item of this list.
(Image 1)
After selecting the file during the importing process, this dialogue appears.
(Image 2)
It should only appear for .csv imports (which have multiple columns, separated by commas), but it is currently appearing for .txt files too, which do not have multiple columns. Maybe another bug: If you press cancel you can't cancel the import all together anymore, and now the list is just empty
(Image 3)
The results of the import show that scratch is considering this file to be a .csv file with the same data, as seen when the same .txt file is parsed as a .csv file.
(Image 4)
This is important to fix, because many scratchers will be importing .txt files with commas, and they want to keep the whole line. If they do want to import a 2D array, they could just switch the file type to .csv, so that both file types work as intended.
{% endraw %}
