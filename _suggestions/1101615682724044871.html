There's a setting for this addon that allows you to preview a custom block's definition comment by hovering on it. The idea for this addon is somewhat similar, but with variables instead of custom blocks.
If there is a reporter of a variable outside of any other reporters with a comment attached, then any block using that variable should show the comment when hovering over it (example in attached image).
Something I'd like to note is that this setting would interfere with the Clean Up Blocks+ dev tools setting, as it deletes any orphaned reporter blocks. Maybe that setting should ignore any reporters with comments attached?
